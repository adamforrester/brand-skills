# Design — `.brand/manifest.json` + `.brand/.health.json`

**Status:** approved 2026-06-10 (brainstorm complete; ready for implementation plan)
**Tasks closed:** [#2, #6](../../tasks.md)
**Tasks unblocked on land:** #3, #4, #5

This spec defines two new machine-readable artifacts in `.brand/`:

- `manifest.json` — facts emitted by `/brand-context:extract` about what files exist, with what status, from which extraction stage.
- `.health.json` — verdict emitted by `/brand-context:check` (and `brand-cli score`) about the package's overall readiness for embedded use.

Hosts (orchestrators, MCP servers, CI gates) consume these files to make decisions without human review. Conversational flows for human practitioners stay unchanged.

---

## 1. Status vocabulary

Shared by both files. Defined once.

| Status | Meaning | How it's set |
|---|---|---|
| `complete` | Schema sections populated with real content | Body ≥ 50 chars after stripping frontmatter, H1, comments; placeholder marker absent; uncommented frontmatter values present. Existing `score.js` `hasContent()` logic. |
| `partial` | Has content but obvious gaps | Some sections populated; others have TODO markers, empty frontmatter, or known-required sections missing. |
| `placeholder` | Scaffold only | Contains `<!-- Fill this file following the schema… -->` marker, OR all frontmatter values are commented (`#`) and body < 50 chars. |
| `missing` | File doesn't exist | `existsSync(path) === false`. |
| `defaults` | Content present but inferred/generic; trust degraded | The writing stage declared low confidence at emit time (e.g., voice from <10 web samples; tokens from a single-page sample; overview from screenshots only with no PDF). |

### Status detection responsibility

Two of these statuses can only be set by the producer:

- `partial` — requires per-file knowledge of expected sections. Producer (extract) sets it from stage-execution data ("Stage 3 wrote voice.md but stopped at 5 samples"). Pure file-content scan can't reliably tell `partial` from `complete`.
- `defaults` — requires confidence signal from the writing stage. Pure file-content scan can't see it.

The other three (`complete`, `placeholder`, `missing`) are set by file-content scan and shared by both code paths.

**Implication:** when `check` runs without `.brand/manifest.json` present, every file is scored as one of `{complete, placeholder, missing}` — never `partial` or `defaults`. The `confidence` field in `.health.json` caps at MEDIUM to reflect the lost signal. When `manifest.json` is present, `check` mirrors the producer-set statuses verbatim.

## 2. `manifest.json`

**Path:** `.brand/manifest.json`
**Owner:** `/brand-context:extract`. CLI helper: `brand-cli emit-manifest`.
**Policy:** Overwrite wholesale on every extract run. Same policy as `design.md` and `brand.md`.

### Shape

```json
{
  "_comment": "Generated by brand-cli. Do not hand-edit — overwritten on every /brand-context:extract run.",
  "version": "1",
  "generated_at": "2026-06-10T14:23:11Z",
  "generator": "brand-cli@0.4.0",
  "tier": "standard",
  "client": "acme",
  "files": {
    "overview.md":             { "status": "complete",    "bytes": 4218 },
    "voice.md":                { "status": "partial",     "bytes": 1872, "note": "observed_voice populated; prescriptive sections empty" },
    "tokens/colors.md":        { "status": "complete",    "bytes": 1340 },
    "tokens/typography.md":    { "status": "complete",    "bytes": 980 },
    "tokens/spacing.md":       { "status": "defaults",    "bytes": 612,  "note": "single-page sample; confidence LOW" },
    "tokens/motion.md":        { "status": "placeholder", "bytes": 96 },
    "tokens/surfaces.md":      { "status": "complete",    "bytes": 770 },
    "composition/page-types.md":   { "status": "missing" },
    "composition/patterns.md":     { "status": "missing" },
    "composition/anti-patterns.md":{ "status": "missing" },
    "conflicts.md":            { "status": "complete",    "bytes": 312 },
    "CHANGELOG.md":            { "status": "missing" }
  },
  "stages": {
    "1_figma":     { "ran": false, "reason": "skipped: sources.figma unset" },
    "2_web":       { "ran": true,  "wrote": ["tokens/colors.md","tokens/typography.md","tokens/spacing.md","tokens/surfaces.md"], "confidence": "MEDIUM" },
    "3_voice":     { "ran": true,  "wrote": ["voice.md"], "samples": 14, "confidence": "MEDIUM" },
    "4_overview":  { "ran": true,  "wrote": ["overview.md"], "sources": ["pdf:brand-guide.pdf","screenshots:3"] },
    "5_conflicts": { "ran": true,  "wrote": ["conflicts.md"], "active": 0 },
    "6_components":{ "ran": false, "reason": "skipped: tier != comprehensive" },
    "8_brand_md":  { "ran": true,  "wrote": ["../brand.md","../design.md"] }
  },
  "mcps": {
    "playwright":    { "available": true,  "used": ["2_web","3_voice"] },
    "figma_console": { "available": false, "used": [] }
  }
}
```

### Field rules

| Field | Type | Rules |
|---|---|---|
| `_comment` | string | Optional. Hand-edit warning. |
| `version` | string | `"1"` to start. Bump on breaking schema changes. |
| `generated_at` | string | ISO 8601 UTC. |
| `generator` | string | `brand-cli@<semver>` when CLI emits; `brand-extract-skill@<plugin-version>` when SKILL fallback emits. Lets hosts diagnose drift between CLI and SKILL. |
| `tier` | enum | `minimum` / `standard` / `comprehensive`. Mirrored from `.brandrc.yaml`. |
| `client` | string | Mirrored from `.brandrc.yaml`. |
| `files` | object | Keyed by path relative to `.brand/`. Tier-aware (see below). Each entry: `status` (required), `bytes` (when file exists), optional `note` ≤120 chars. |
| `stages[*].wrote` | array | Paths relative to `.brand/`. Project-root files (e.g., `brand.md`, `design.md`) prefix with `../`. |
| `stages` | object | Keyed by `<n>_<slug>` matching SKILL's stage numbering: `1_figma`, `2_web`, `3_voice`, `4_overview`, `5_conflicts`, `6_components`, `8_brand_md`. (No Stage 7 — historical, see CLAUDE.md.) Each entry: `ran` (required bool); when `ran: true`: `wrote: string[]`; when `ran: false`: `reason: string`. Optional per-stage extras: `confidence`, `samples`, `active`, `sources`. |
| `mcps` | object | Keyed by MCP name: `playwright`, `figma_console`, etc. Each entry: `available: bool`, `used: string[]` (stage keys). Critical input for #3 (per-stage MCP-fallback contract). |

### Tier-aware `files` map

- Files relevant to the active tier always appear.
- Out-of-tier files that *do* exist also appear (so a hand-written `composition/patterns.md` at `tier: minimum` is surfaced rather than hidden).
- `components/*.md` listed individually only at `tier: comprehensive`.

## 3. `.health.json`

**Path:** `.brand/.health.json` (dotfile prefix because it's a verdict, not source).
**Owner:** `/brand-context:check`. Also written by `brand-cli score`.
**Policy:** Overwrite wholesale on every check/score run.

### Shape

```json
{
  "_comment": "Generated by brand-cli score. Do not hand-edit.",
  "version": "1",
  "generated_at": "2026-06-10T14:25:42Z",
  "generator": "brand-cli@0.4.0",
  "tier": "standard",
  "client": "acme",
  "readiness": 0.50,
  "tier_label": "partial",
  "manifest_seen": true,
  "manifest_generated_at": "2026-06-10T14:23:11Z",
  "weights": {
    "overview.md": 2,
    "voice.md": 2,
    "tokens/colors.md": 1,
    "tokens/typography.md": 1,
    "tokens/spacing.md": 1,
    "tokens/motion.md": 1,
    "tokens/surfaces.md": 1,
    "composition/page-types.md": 1,
    "composition/patterns.md": 1,
    "composition/anti-patterns.md": 1,
    "conflicts.md": 1,
    "CHANGELOG.md": 1
  },
  "weighted_complete": 7,
  "weighted_total": 14,
  "confidence": "MEDIUM",
  "files": {
    "overview.md":             "complete",
    "voice.md":                "partial",
    "tokens/colors.md":        "complete",
    "tokens/typography.md":    "complete",
    "tokens/spacing.md":       "defaults",
    "tokens/motion.md":        "placeholder",
    "tokens/surfaces.md":      "complete",
    "composition/page-types.md":    "missing",
    "composition/patterns.md":      "missing",
    "composition/anti-patterns.md": "missing",
    "conflicts.md":            "complete",
    "CHANGELOG.md":            "missing"
  },
  "gaps": [
    { "file": "tokens/motion.md", "status": "placeholder", "suggested_action": "Run /brand-context:extract Stage 2 with Playwright, or paste motion tokens manually" },
    { "file": "composition/page-types.md", "status": "missing", "suggested_action": "Hand-author per schema/brand/composition-page-types.schema.md" },
    { "file": "composition/patterns.md", "status": "missing", "suggested_action": "Hand-author per schema/brand/composition-patterns.schema.md" },
    { "file": "composition/anti-patterns.md", "status": "missing", "suggested_action": "Hand-author per schema/brand/composition-anti-patterns.schema.md" },
    { "file": "CHANGELOG.md", "status": "missing", "suggested_action": "Created automatically on next /brand-context:extract" }
  ],
  "downgrades": [
    { "file": "tokens/spacing.md", "reason": "single-page sample; confidence LOW (from manifest)" }
  ]
}
```

### Field rules

| Field | Type | Rules |
|---|---|---|
| `readiness` | number 0.0–1.0 | Two-decimal precision. Formula below. |
| `tier_label` | enum | Derived from `readiness`. `>=0.95` → `ready`. `>=0.80` → `good`. `>=0.50` → `partial`. `<0.50` → `incomplete`. |
| `manifest_seen` | bool | `true` if `.brand/manifest.json` was present and used. `false` means content-scan fallback. |
| `manifest_generated_at` | string | Echoed from manifest. Allows hosts to detect stale verdicts. Omitted when `manifest_seen: false`. |
| `weights` | object | Echo of the table used. Hosts can reproduce the math without owning a copy. Source: `cli/src/utils/tier-weights.js`. |
| `weighted_complete` | integer | Σ weights for files where status ∈ {`complete`, `defaults`}. |
| `weighted_total` | integer | Σ weights for every required file in tier. |
| `confidence` | enum | `HIGH` when `manifest_seen` and no `defaults` files. `MEDIUM` when `manifest_seen` with `defaults` present, OR manifest absent and ≥80% files `complete` by content scan. `LOW` otherwise. Mirrors `audits.schema.md`. |
| `files` | object | Flat `{path: status}` map. Compact mirror of `manifest.files[*].status`. |
| `gaps` | array | Every file with status ∈ {`partial`, `placeholder`, `missing`}. Each entry: `{file, status, suggested_action}`. Action text from per-status / per-path lookup table (matches existing `brand-check/SKILL.md` Step 3). |
| `downgrades` | array | Every file with status `defaults`. Each entry: `{file, reason}`. Empty when none. Empty when `manifest_seen: false`. |

### Readiness formula

```
weighted_complete = Σ (weight[file] for file where status ∈ {complete, defaults})
weighted_total    = Σ (weight[file] for every required file in tier)
readiness         = weighted_complete / weighted_total
```

### Initial weight table (hardcoded in `cli/src/utils/tier-weights.js`)

| File | Weight |
|---|---|
| `overview.md` | 2 |
| `voice.md` | 2 |
| `tokens/{colors,typography,spacing,motion,surfaces}.md` | 1 each |
| `composition/{page-types,patterns,anti-patterns}.md` | 1 each |
| `conflicts.md` | 1 |
| `CHANGELOG.md` | 1 |
| `workflows/{figma-to-code,code-standards,deploy,qa-checklist}.md` | 1 each (comprehensive only) |
| `components/*.md` | not weighted individually; presence/absence reported in `gaps` at comprehensive tier |

Practitioner-tunable weights are out of scope.

### Coupling rules

- `.health.json` MAY exist without `manifest.json` (practitioner ran `check` before `extract`). `manifest_seen: false`, `downgrades: []`, `confidence` capped at MEDIUM.
- `manifest.json` MAY exist without `.health.json` (`extract` ran but `check` never has). `extract` does NOT synthesize a health file.
- When both exist, `health.manifest_generated_at === manifest.generated_at` if cache-coherent. If different, manifest is fresher; host should re-run `check`.

## 4. File-write policies, `.gitignore`, init interactions

### CLAUDE.md additions to "File-write policies — additive vs. overwrite" table

| File | Policy | Why |
|---|---|---|
| `manifest.json` | **Overwrite wholesale every run** | Generated artifact; source of truth is `.brand/*.md`. Same as `design.md`/`brand.md`. |
| `.health.json` | **Overwrite wholesale every run** | Verdict cache. Reproducible from manifest + tier weights. |

### `.gitignore` stance

**Default: commit both.** They're useful in version control:
- `manifest.json` shows extraction state at commit time.
- `.health.json` shows readiness history (PR reviewer can see "this PR pushed readiness 0.62 → 0.81").
- Hosts that consume the package via git checkout get them for free.

`init.js` does NOT add either to `.gitignore`. Practitioners who want them ignored add it themselves.

### init scaffolding

`brand-cli init` does NOT scaffold either file. Both appear only after the relevant command runs. A fresh project has neither — that's the signal that extract/check haven't run yet. (`manifest_seen: false` becomes a meaningful host signal.)

### Interaction with `.brand/audits/`

Distinct concern. `audits/` is per-target adherence audits (output of `/brand-context:audit`). These two new files are package-state files. They live alongside but don't interact. A future enhancement could let an audit's `audit_confidence` read `.health.json.confidence` — out of scope here.

### Partial-pipeline behavior

If `extract` runs and only some stages succeed, manifest still emits — partially-`ran: true`, others `ran: false` with reasons. **A partial pipeline still produces a valid manifest.**

## 5. JSON Schema validation

### New files

```
schema/
├── manifest.schema.json     ← NEW: JSON Schema 2020-12 for manifest.json
├── health.schema.json       ← NEW: JSON Schema 2020-12 for .health.json
└── brand/
    └── README.md            ← UPDATED: cross-link to ../manifest + ../health schemas
```

Why at `schema/` root, not `schema/brand/`: those are markdown specs for `.brand/*.md` shape. These are machine-validation contracts for the toolkit. Distinct.

### Schema highlights

`manifest.schema.json`:
- Required: `version`, `generated_at`, `generator`, `tier`, `files`, `stages`, `mcps`.
- `version: { const: "1" }`.
- `generator: { type: "string", pattern: "^(brand-cli|brand-extract-skill)@" }`.
- `files`: `additionalProperties` is an object with required `status` (enum: complete/partial/placeholder/missing/defaults), optional `bytes` (integer, ≥0), optional `note` (string, ≤120).
- `stages`: `patternProperties` keyed `^[1-8]_[a-z_]+$`. Each: required `ran` (bool); optional `wrote`, `reason`, `confidence` (HIGH/MEDIUM/LOW), `samples`, `active`, `sources`.
- `mcps`: `additionalProperties` is an object with required `available` (bool), `used` (array of strings).
- `additionalProperties: false` at root, except `_comment` declared as optional string.

`health.schema.json`:
- Required: `version`, `generated_at`, `generator`, `tier`, `readiness`, `tier_label`, `manifest_seen`, `weights`, `weighted_complete`, `weighted_total`, `confidence`, `files`, `gaps`, `downgrades`.
- `readiness: { type: "number", minimum: 0, maximum: 1 }`.
- `tier_label: { enum: ["ready", "good", "partial", "incomplete"] }`.
- `gaps[]`: required `{file, status, suggested_action}`.
- `downgrades[]`: required `{file, reason}`.

### Validation behavior

- **CLI path** (`brand-cli emit-manifest`, `brand-cli score`) compiles the schema with `ajv` + `ajv-formats` and validates BEFORE writing. Throws on violation. Fail-loud.
- **SKILL fallback path** does NOT validate. The SKILL is the spec in prose; bugs there are SKILL bugs to fix.

### `package.json` additions

```diff
"dependencies": {
+ "ajv": "^8",
+ "ajv-formats": "^3"
}
```

Two small, stable, widely-used deps. Acceptable cost for the validation guarantee.

## 6. SKILL → CLI fallback contract

Per CLAUDE.md, every CLI command has a SKILL.md inline fallback. That contract holds here.

### `brand-cli emit-manifest` (new)

Single-purpose CLI command. Accepts stage-execution data on stdin as JSON; emits `.brand/manifest.json`.

```bash
echo '{"stages": {...}, "mcps": {...}}' | brand-cli emit-manifest
echo '{"stages": {...}, "mcps": {...}}' | brand-cli emit-manifest --dry-run  # to stdout
```

The CLI walks `.brand/`, classifies each file via shared `file-status.js`, folds in the stdin stage-execution data, validates against schema, writes the file (or stdout in dry-run).

### Why SKILL drives + CLI helps

`extract` is SKILL-owned (it walks the practitioner through a multi-stage conversational flow). The CLI exists to centralize deterministic regen — including emitting valid JSON. Splitting it as: SKILL accumulates `stages` data, then shells out to `brand-cli emit-manifest` to write the file with validation. Inline fallback (CLI absent): SKILL constructs JSON directly and `Write`s without validation.

### `brand-cli score` (existing, extended)

Currently emits human console output and an optional `--json` stdout. Always emit `.brand/.health.json` to disk in addition.

SKILL fallback (CLI absent): `brand-check/SKILL.md` constructs the same JSON inline and `Write`s it.

## 8. Testing

The repo has no test harness today (`npm test` is a stub). Not introducing one as part of this task. Verification protocol:

### Manual end-to-end walks

- Populated `.brand/` → run `brand-cli score` → confirm `.health.json` written, validates, expected `readiness`.
- Fresh `brand-cli init` directory → run `score` → confirm `readiness < 0.1`, all files in `gaps`.
- Partial `.brand/` (mix of complete + placeholder) → confirm tier-weighted math hand-checks.
- Crafted stdin → `brand-cli emit-manifest` → validates → round-trip through `score` produces `.health.json` with `manifest_seen: true` and matching `confidence`.

### JSON Schema self-validation

`cli/test/validate-schemas.js` (run via `npm run lint:schemas`):
- Loads both schemas through ajv (catches malformed schemas)
- Validates `tests/golden/manifest.json` against `schema/manifest.schema.json`
- Validates `tests/golden/health.json` against `schema/health.schema.json`
- Validates an intentionally-broken payload, expects rejection

Two golden fixtures committed. They double as integration reference for hosts.

### Round-trip integration check

```bash
cd tmp-test && brand-cli init && brand-cli score
# expect: .brand/.health.json exists, validates, readiness ≈ 0.05
```

## 9. Out of scope

Explicitly excluded; deferred or rejected during brainstorm:

- **Per-section status drilldown** inside files (rejected; per-file is the granularity).
- **Practitioner-tunable file weights** (hardcoded in `tier-weights.js`).
- **SHA-256 hashes per file in manifest** (rejected; `bytes` is enough; git owns change detection).
- **Schema versioning beyond `version: "1"`** (when v2 is needed, write a migration note).
- **`extract` becoming a CLI command** (stays SKILL-owned; CLI exposes `emit-manifest` only).
- **JSON Schema for `.brandrc.yaml`** (separate, valuable but not blocking).
- **Per-stage MCP-fallback contract** ([#3](../../tasks.md)) — manifest schema accommodates it via `stages[*].reason`, but the contract of what each stage emits is its own task.
- **`.scope.json` as alternate input** ([#4](../../tasks.md)) — separate task, separate file.
- **Static `schema/tier-weights.json` publication** (rejected; weights echoed inline in every `.health.json`).
